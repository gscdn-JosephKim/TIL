사내 WiFi 는 SSID 가 숨김 처리 되어 있습니다.
그렇기 때문에 공개된 WiFi SSID 리스트에서 선택한 뒤 암호를 입력하는 방식으로는 사용할 수 없습니다.

각 기기 별 방법을 모두 공유드리긴 어렵지만 대략 다음과 같이 접속하실 수 있습니다.


+ WiFi 접속 메뉴에서 숨겨진 SSID 를 선택하기 위해 다음을 따라 선택하시면 됩니다.
 - Windows : 숨겨진 네트워크 선택
 - 삼성 android : 네트워크 추가 선택
 - iOS : 기타 선택
 - Mac : 다른 네트워크에 연결 (Join Other Network) 선택

+ 보안은 다음 중 일치 하는 것을 선택하시면 됩니다.
 - WPA/WPA2-Personal 과 같은 형태를 선택
 - WPA2/WPA3 Personal 과 같은 형태를 선택
 - WPA2/WPA3 과 같은 형태를 선택

+ 내부인이 접속하는 wifi 암호 : 절대 직원분들 외에는 알려주시면 안됩니다.
 - SSID : 모두 소문자 : [[SSID]]
 - 암호 : 대소문자 혼합 : [[PASSWORD]]

+ 외부인에게 알려주는 wifi 암호
 - SSID : 모두 소문자 : [[SSID]]
 - 암호 : 대소문자 혼합 : [[PASSWORD]]

+ 참고 : 자동으로 접속하게 하려면 다음을 꼭 선택해주세요.
 - 삼성 android : 자동으로 다시 연결 선택
 - Windows : 자동으로 연결 선택

———————————————+++ 접속이 안되는 이유 +++———————————————
WiFi 접속시 접속을 방해하는 방법이 있습니다.
WiFi deauth 라는 공격방법인데, 정상적으로 접속 하려는 WiFi 인증과정을 방해하는 해킹 기법입니다.

요즘에는 이 해킹 기법을 활용하여 금융회사 또는 중요한 개인정보를 취급하는 곳에서는 
비 인가된 WiFi 장비를 설치하거나, 휴대폰 tethering 생성등을 막기 위해 관련 기술을 사용하는 장비를 도입하는 경우가 많습니다.
해킹 기법이 데이터 유출을 막기 위한 보안 기법으로 발전하게 된 사례죠.

자세한 내용은 wikipedia 의 링크를 붙여드립니다. 관련된 키워드로 검색을 해보시면 다양한 자세한 내용들을 검색하실 수 있습니다.
https://en.wikipedia.org/wiki/Wi-Fi_deauthentication_attack



———————————————+++ 왜 불편하게 SSID 를 숨겼는가? +++———————————————
이런저런 방법을 찾아보다가 SSID 를 숨겨두게 되면 WiFi deauth attack 의 대상 리스트에서 벋어날 수도 있겠다 싶었습니다.
기술적으로 여러 이슈가 있는데 숨겨진 SSID 자체를 찾아내기 위해서는 무한 대입을 통해서만 숨겨진 SSID 의 이름을 찾을 수 있고, 
그런 이유로 숨겨진 SSID 를 사용 하라는 내용을 보게 되었습니다.

그래서 테스트 하여 확인해본 결과 접속에 문제가 되는 경우가 현저히 줄어들었고, 
문제가 해결되었다고 생각하여 SSID 를 숨기는 결정을 하게 되었습니다.

참고로 한번에 접속이 안되면 여러번 시도하시면 제 경우에는 잘 접속되었습니다.
